# Morning (Daily List)

## Trigger

- 今日待办
- 今天要做什么
- 帮我生成今天清单

## Goal

生成「今日轻量待办清单」（3–7 条），用于当天执行。
默认只在对话中输出，不写入 flomo。

## Input

- flomo 规则 memo（状态标签/工作流/识别规则/预约型规则等）
- 近 14 天的：#状态/待办 / #状态/待跟进
- 可能包含时间/截止/提醒的预约型 memo

## Steps

1. 读取规则 memo（若可用）
2. 搜索候选 memo：
   - 普通待办：近 14 天的待办/待跟进
   - 预约型：包含日期/截止/提前提醒的记录
3. 预约型触发判断：
   - 仅当“今天是触发日/已过触发日且未完成”才纳入
4. 合并与去重：
   - 同一事项合并为一个行动
5. 筛选与压缩（控制 3–7 条）：
   - 优先：截止/已触发 > 卡住的待跟进 > 最近新增重要项
6. 生成清单（结构化）
7. 结尾询问是否确认当天计划

## Output

每条事项包含：
- 事项（一句话）
- 来源（memo_id/引用）
- 当前状态（待办/待跟进）
- 目标日期/触发规则
- 下一步动作（可立即执行）
- 优先级（高/中/低）
- 是否需要确认（是/否）

## Write Policy

- 默认只读
- 仅当用户明确要求（“保存今天清单/写入 flomo”）时创建新 memo
- 不修改原 memo，不添加状态标签到清单 memo

## Notes

- 不输出超过 7 条
- 不解释规则与中间推理
- 保持简洁但可执行
