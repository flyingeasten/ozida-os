# Detect (Todo Triage)

## Trigger

- 帮我整理这些 memo
- 哪些是待办
- 这些需要我做吗
- 标记一下状态

## Goal

从 flomo memo 中识别：
- 状态/待办（我需要主动做）
- 状态/待跟进（需要我追踪）
- 状态/待整理（仅记录/待整理）

并给出标签建议（不自动写入）。

## Input

- 最近一段时间的 flomo memo（默认 7–14 天）
- 现有标签：#状态/待办 / #状态/待跟进 / #状态/待整理 等

## Steps

1. 扫描 memo，识别是否包含“可执行动作”或“需要跟进”的语义
2. 分类：
   - 待办：包含明确动作（发/做/改/联系/跟进…）
   - 待跟进：依赖外部结果（等客户/等供应商/等审批…）
   - 待整理：信息记录/想法/会议纪要
3. 为每条 memo 生成：
   - 摘要
   - 建议标签
   - 判断理由
4. 汇总输出清单（不写回）

## Output

对每条候选 memo：
- 摘要
- 建议标签
- 理由
- 是否需要确认：是

## Write Policy

- 默认只读
- 不自动添加标签
- 仅在用户明确确认后执行标签添加

## Notes

- 不将模糊或无行动价值的记录强行归类
- 不修改原始内容
