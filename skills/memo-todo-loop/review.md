# Review (Closeout)

## Trigger

- 今天做完了吗
- 收尾一下
- 更新今日状态

## Goal

基于用户反馈生成“状态修改方案”，并在确认后更新 flomo 状态标签。

## Input

- 当天清单（本次对话优先）
- 用户反馈（编号/自然语言）

## Steps

1. 回顾今日清单（优先使用本次对话的 morning 结果）
2. 解析用户反馈：完成 / 继续 / 待跟进 / 搁置 / 取消
3. 映射状态：
   - 完成 → 状态/已完成
   - 继续 → 状态/待办（保持）
   - 待跟进 → 状态/待跟进
   - 搁置 → 状态/搁置
   - 取消 → 状态/取消
4. 生成拟修改方案（逐条）：事项 / memo_id / 当前→建议 / 理由
5. 安全检查（来源不清/多条对应/高风险）
6. 等待用户确认
7. 确认后执行更新（逐条）

## Output

- 修改方案清单
- 执行后结果（修改前 → 修改后）

## Write Policy

- 未确认前不执行任何修改
- 不删除 memo
- 不批量误改历史记录

## Notes

- 状态标签表示“当前阶段”，应定期清理
- 保留原始内容，不改写文本
